# Agent Instructions

## 提交前要求

每次提交前，都必须在 `docs/record/` 下新增一份以当前时间命名的 Markdown 变更记录文件。

文件名建议使用 `YYYY-MM-DD-HHMMSS.md`，例如 `2026-05-10-195423.md`。

每份变更记录必须包含以下内容：

1. `为什么变更`：说明变更背景、目标和要解决的问题。
2. `变更内容`：概括实际修改的代码、配置、文档、测试或运行文件。
3. `可能存在的隐患`：列出潜在风险、未验证点、兼容性问题、运行态副作用或提交边界风险。
4. `后续方向建议`：给出后续验证、拆分、重构、产品化或运维建议。

生成记录前应先查看真实工作区差异，不能只根据记忆或口头描述编写。若工作区包含不应提交的运行产物、缓存、密钥或本地数据库文件，必须在记录中明确指出，并在提交前单独处理。

每次提交前还必须通过全部代码测试。默认执行项目完整测试命令；如果因环境、依赖、外部服务或硬件限制无法完整运行，必须在变更记录和提交说明中明确写出未运行的测试、原因、已执行的替代验证，以及剩余风险。不能在测试失败或未说明测试缺口的情况下提交。
