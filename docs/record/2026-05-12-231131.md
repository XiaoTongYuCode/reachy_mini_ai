# 2026-05-12 23:11:31 变更记录

## 为什么变更

本地 `main` 分支已有 1 个待推送提交，推送过程中长时间无输出。检查真实提交差异后发现该提交误纳入了 `services/hf_realtime_gateway/mlx_models/` 下的本地模型缓存，其中 `weights.npz` 约 2.9G，不适合进入普通 GitHub 仓库历史，也可能导致远端拒绝推送。

本次变更目标是在不删除本地模型文件的前提下，清理待推送提交边界，避免把大模型权重、下载缓存或运行态产物发布到远端。

## 变更内容

- 从 Git 索引中移除 `services/hf_realtime_gateway/mlx_models/`，保留本地工作区文件用于本机运行。
- 在 `.gitignore` 中新增 `services/hf_realtime_gateway/mlx_models/` 忽略规则，降低后续误提交模型权重的风险。
- 为 `services/hf_realtime_gateway` 子项目 dev 依赖补充 `pytest-asyncio` 并更新 `uv.lock`，保证子项目单独运行 async 测试时依赖完整。
- 新增本变更记录，说明本次推送前的提交边界清理、风险和验证结果。

## 可能存在的隐患

- 本地模型目录被保留但不再进入版本库；其他机器需要按文档或启动逻辑自行下载、放置或配置 Hugging Face / MLX 模型文件。
- 原始本地提交历史曾包含 2.9G 权重对象，后续需要通过 amend 改写未推送提交后再推送，避免远端接收该对象。
- 已执行根项目与子项目测试；测试不覆盖真实 Hugging Face 语音模型下载、真实麦克风/扬声器链路或真实 Reachy Mini 硬件运行。

## 后续方向建议

- 将大模型缓存、下载目录和运行态产物统一放入明确的本地缓存目录，避免散落在服务源码目录下。
- 如果模型权重需要团队共享，应使用 Hugging Face Hub、对象存储、Git LFS 或发布脚本，而不是普通 Git blob。
