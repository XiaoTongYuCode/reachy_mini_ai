# 2026-07-03 10:45:10 变更记录

## 为什么变更

本次提交整理 Aliyun DashScope Realtime 后端接入，使 `BACKEND_PROVIDER=aliyun` 成为独立实时语音后端，并避免 Aliyun 模型配置继续复用 OpenAI/Gemini 的通用 `MODEL_NAME`。同时，Aliyun 原生 websocket 支持模型侧 function calling，需要把现有本地 tools、相机视觉帧和后台任务管理接到同一条链路，避免 camera 长时间视觉序列阻塞工具调用。

## 变更内容

- 新增 `src/reachy_mini_conversation_app/aliyun_realtime.py`，实现 Aliyun DashScope native realtime websocket 会话、session 配置、音频输入输出、工具调用结果回传、原生 `input_image_buffer.append` 视觉帧发送和基础成本统计。
- 配置层新增 `ALIYUN_BACKEND`、`DASHSCOPE_API_KEY`、`ALIYUN_REALTIME_MODEL`、Aliyun websocket URL、采样率和视觉帧窗口配置；`MODEL_NAME` 只继续服务 OpenAI/Gemini，避免跨供应商模型名污染。
- 启动、控制台和前端静态页增加 Aliyun backend 的选择、凭据保存、状态判断和提示文案。
- `camera` 工具新增 `duration_seconds`，仅在 Aliyun backend 下请求 1 FPS 视觉序列；新增 `BackgroundToolManager.start_coroutine_tool()` 支持内部 coroutine 后台工具。
- 新增 `cancel_aliyun_camera_sequence` 系统工具，并把它加入系统工具常量和文档，支持取消最新或指定的 Aliyun camera sequence。
- 更新 `.env.example`、`README.md`、`README_zh.md`，说明 Aliyun 配置、视觉序列、取消方式，以及联网搜索、天气、SMTP 等工具配置与 backend 解耦。
- 新增/扩展 Aliyun realtime、配置隔离、camera duration、sequence cancel 相关测试。
- 本次提交会包含上一份未跟踪记录 `docs/record/2026-07-02-232545.md`，并按提交前要求新增当前记录 `docs/record/2026-07-03-104510.md`。
