# 2026-05-10 19:54:23 变更记录

## 为什么变更

当前工作区的改动主要围绕 Reachy Mini 的对话能力继续增强：一方面让实时语音后端能够利用本地持久化记忆，减少跨会话上下文丢失；另一方面补充邮件发送、显式记忆管理、模型可见消息日志等工具能力，便于调试和实际使用。同时，默认交互语言和提示词被调整为中文，配合语音转写语言设置改为中文。

## 变更内容

1. 本地记忆能力
   - 新增 `src/reachy_mini_conversation_app/storage/memory.py`，用 SQLite + FTS5 保存会话、消息和显式长期记忆。
   - 支持历史消息搜索、长期记忆搜索、邻近上下文扩展、重要记忆注入 session instructions，以及归档/遗忘能力。
   - 在 `main.py` 创建 `MemoryStore`，并通过 `ToolDependencies.memory_store` 注入运行时。
   - 在 OpenAI / Hugging Face 共用的 `BaseRealtimeHandler` 和 Gemini Live handler 中加入记忆会话创建、最终转写落库、相关历史检索和模型上下文刷新。

2. 显式记忆工具
   - 新增 `src/reachy_mini_conversation_app/tools/manage_memory.py`。
   - 新增 `SystemTool.MANAGE_MEMORY`，让 `manage_memory` 作为系统工具自动加载，不依赖单个 profile 的 `tools.txt`。
   - 工具支持 `remember`、`update`、`forget`、`search` 四类动作。

3. 邮件发送工具
   - 新增 `src/reachy_mini_conversation_app/tools/send_email.py`，通过 SMTP 发送纯文本或 HTML 邮件。
   - 支持 `SMTP_*` 通用环境变量，也兼容 `GMAIL_EMAIL` / `GMAIL_APP_PASSWORD`。
   - `.env.example` 增加 SMTP 配置示例和 `default_target_email`。
   - 多个 profile 的 `tools.txt` 加入 `send_email`，让默认角色可调用邮件工具。

4. 实时后端与可观测性
   - 新增 `src/reachy_mini_conversation_app/agent_observability.py`，记录模型可见的 message list，并对内联图片数据做脱敏。
   - OpenAI / Hugging Face / Gemini 路径都在关键边界记录用户转写、助手转写、工具结果、相机输入和响应创建上下文。
   - OpenAI / Hugging Face 的输入转写语言从 `en` 改为 `zh`。
   - Gemini 路径新增记忆上下文发送与日志记录，并保留对不支持 `language_codes` 的测试覆盖。

5. 中文默认交互
   - `default_prompt.txt`、基础身份、机智身份、沉默机器人行为和龙虾笑话提示被翻译/调整为中文。
   - 默认提示词明确“默认使用中文回答”，回答长度也按中文表达重新约束。

6. 测试覆盖
   - 新增 `tests/test_memory.py` 覆盖 SQLite 初始化、旧库迁移、中文搜索、历史片段注入、记忆工具操作和上下文预算。
   - 新增 `tests/test_agent_observability.py` 覆盖 message list 日志、图片脱敏、单轮只打印一次和 scoped message 替换。
   - 新增 `tests/tools/test_send_email.py` 覆盖 SMTP 默认收件人、HTML 邮件、缺少收件人和认证失败。
   - 扩展 `tests/test_openai_realtime.py` 和 `tests/test_gemini_live.py` 覆盖转写落库、记忆上下文刷新和 Gemini 输入转写配置约束。

## 可能存在的隐患

1. `memory.sqlite3`、`memory.sqlite3-shm`、`memory.sqlite3-wal` 已移动到 `src/reachy_mini_conversation_app/storage/` 并通过 `.gitignore` 忽略；这些是本地运行态数据，不应随代码提交。
2. `storage/memory.py` 较大，承担了 schema、迁移、搜索、评分和格式化多类职责，后续维护时可能需要继续拆分边界。
3. Gemini 的记忆上下文目前通过 `send_realtime_input(text=...)` 注入，语义上不同于 OpenAI/HF 的 session instructions 更新，实际模型行为仍需做端到端验证。
4. 邮件工具会真实发送外部邮件，虽然工具描述要求显式用户请求后才使用，但仍建议在系统提示或运行策略中继续强调外部可见动作需要确认。
5. 中文搜索依赖自定义分词/ngram 和打分阈值，短查询或噪声 ASR 场景可能出现召回不足或误召回。
6. 当前变更范围横跨 prompt、工具、持久化、实时后端和测试，提交前需要确认是否按功能拆分提交，避免一次提交过大。

## 后续方向建议

1. 在 `.gitignore` 中忽略本地记忆数据库运行产物，或明确给出是否允许提交空库文件的项目规则。
2. 跑一遍聚焦测试：`tests/test_memory.py`、`tests/test_agent_observability.py`、`tests/tools/test_send_email.py`、`tests/test_openai_realtime.py`、`tests/test_gemini_live.py`。
3. 用 mockup sim 或本地实时后端做一次中文语音链路验证，重点看转写语言、记忆检索注入和 message list 日志是否符合预期。
4. 将 `MemoryStore` 继续拆成 schema/storage、retrieval、context formatting 三层，降低后续修改搜索策略时的影响面。
5. 为 `send_email` 增加更明确的运行期确认策略或 dry-run 模式，降低误发风险。
