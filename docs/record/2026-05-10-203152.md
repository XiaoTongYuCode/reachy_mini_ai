## 变更内容

1. 存储架构调整
   - 新增 `src/reachy_mini_conversation_app/storage/` 包。
   - 将原记忆实现迁移为 `src/reachy_mini_conversation_app/storage/memory.py`。
   - 新增 `storage/__init__.py` 导出 `MemoryStore` 和 `MemoryContextProvider`。
   - 更新 OpenAI、Hugging Face、Gemini、主入口和测试中的记忆模块导入路径。
   - `MemoryStore()` 默认把 SQLite 放在 `src/reachy_mini_conversation_app/storage/`。

2. 运行态文件处理
   - 将本地 `memory.sqlite3*` 运行产物从项目根目录移动到 storage 目录。
   - `.gitignore` 增加 `src/reachy_mini_conversation_app/storage/memory.sqlite3*`，避免提交本地数据库和 WAL 文件。

3. 提交规范
   - 新增 `AGENTS.md`，要求每次提交前必须在 `docs/record/` 生成按时间命名的变更记录。
   - 更新 `AGENTS.md`，要求每次提交前必须通过全部代码测试；无法完整测试时必须说明缺口、原因、替代验证和风险。

4. 功能主体
   - 保留并提交本地长期记忆、历史消息检索、显式记忆工具、SMTP 邮件工具、模型 message list 可观测性、中文默认 prompt 和相关测试。

## 可能存在的隐患

1. `storage/memory.py` 仍然承担 schema、迁移、搜索、打分和上下文格式化等多类职责，后续可以继续拆分。
2. SQLite 默认落入源码包目录虽然满足当前本地运行规范，但打包/安装场景下可能需要进一步明确可写目录策略。
3. Gemini 记忆上下文仍是通过实时输入注入，和 OpenAI/HF 的 session instructions 更新方式不同，需要继续做端到端语音链路验证。
4. 邮件工具是外部可见动作，后续最好补充更强的确认或 dry-run 策略。

## 后续方向建议

1. 将 `MemoryStore` 拆分为 schema/storage、retrieval、context formatting 三层。
2. 为 SQLite 存储路径增加运行配置项，区分开发、本地实例和安装包运行场景。
3. 补充端到端手动验证：mockup sim + 中文语音 + 记忆召回 + 邮件工具确认流程。
4. 在后续提交中继续保持单次提交范围聚焦，避免把架构移动、功能行为和 prompt 文案混在一个过大的变更里。
