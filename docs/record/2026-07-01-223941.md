## 为什么变更

本次变更将 Ark 后端收敛为直接调用火山引擎 Realtime websocket 的语音链路，去除原先额外依赖 OpenAI-compatible sidecar / OpenRouter 模型做本地 function-calling 工具路由的路径。目标是减少 Ark 会话前置判断步骤、降低配置复杂度，并让 `BACKEND_PROVIDER=ark` 的运行语义保持干净直接。

## 变更内容

- `src/reachy_mini_conversation_app/ark_live.py`：移除 Ark sidecar 工具路由、OpenAI SDK 动态调用、工具结果 RAG 回传、响应抑制队列和相关状态，只保留火山引擎 websocket 的 ASR、Chat、TTS、memory 和音频收发逻辑。
- `src/reachy_mini_conversation_app/config.py`：删除 Ark 路径复用 `PIPELINE_LLM_*` / `GATEWAY_LLM_*` 作为工具路由模型配置的读取逻辑。
- `.env.example`、`README.md`、`README_zh.md`：删除 Ark sidecar 配置说明，改为说明 Ark 直接使用火山引擎 realtime websocket 完成识别与回复。
- `tests/test_ark_live.py`：删除 sidecar 工具路由、工具结果回传、响应抑制和 pipeline LLM fallback 相关测试，保留 Ark frame、header、session、ASR、Chat、TTS 和不保留 OpenAI SDK 模块级 wiring 的测试。

工作区差异中仅包含上述代码、配置、文档和测试文件；未发现需要单独处理的未跟踪运行产物、缓存、密钥或本地数据库文件。
